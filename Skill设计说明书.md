# 可视化迭代 Skill 设计说明书

## Skill 名称: FitnessVizAgent — 健身数据可视化迭代智能体

---

## 环节 1: 数据理解 (Data Understanding)

| 项目       | 内容                                                                                                                                                   |
| -------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| **核心逻辑** | 解析数据集结构，提取字段名、类型、分布特征、缺失情况，形成数据画像                                                                                                                    |
| **输入**   | 原始 DataFrame（用户上传的 CSV/Excel 数据）                                                                                                                     |
| **输出**   | 数据画像 JSON（含字段列表、类型映射、数值列统计摘要、分类列唯一值列表、缺失值概况、数据维度）                                                                                                    |
| **实现思路** | 使用 pandas 的 `dtypes`、`describe()`、`nunique()`、`isnull().sum()` 自动扫描全部字段；对数值列计算偏度/峰度判断分布形态（正态/偏态）；对分类列记录唯一值及频次分布；将画像结果存入 session 上下文，供后续图表选择和样式优化环节使用 |

### 关键子步骤

1. 字段类型识别：区分数值列（int/float）与分类列（object/category）
2. 统计摘要：计算数值列的 mean/std/min/max/四分位数
3. 缺失值分析：统计每列缺失数和缺失率
4. 数据质量标记：标记缺失率 > 30% 的列，提示用户注意

## 环节 2: 图表选择 (Chart Selection)

| 项目       | 内容                                                                                                                                                                 |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **核心逻辑** | 根据用户自然语言意图 + 数据画像，匹配最优图表类型和字段映射                                                                                                                                    |
| **输入**   | 用户自然语言指令（文本）+ 数据画像（来自环节 1）                                                                                                                                         |
| **输出**   | 图表规格 JSON（chart_type, x, y, color, title, agg, top_n）                                                                                                              |
| **实现思路** | 采用双层策略确保鲁棒性：(1) **规则层** — 基于关键词匹配图表类型（如"柱状"→bar，"散点"→scatter），通过字段名相似度匹配坐标轴列；(2) **LLM 层** — 将数据画像和用户指令组装成 Prompt 送入大语言模型，要求模型返回结构化 JSON 规格。规则层始终作为 LLM 不可用时的回退方案。 |

### 图表类型映射规则

| 关键词                | 图表类型      | 适用场景     |
| ------------------ | --------- | -------- |
| 柱状/柱形/bar          | bar       | 分类对比     |
| 折线/趋势/line         | line      | 时序或有序趋势  |
| 散点/scatter/scatter | scatter   | 两数值变量相关性 |
| 饼/占比/pie           | pie       | 构成比例     |
| 热力/相关/heatmap      | heatmap   | 多变量相关矩阵  |
| 直方/分布/histogram    | histogram | 单变量分布    |
| 箱线/box             | box       | 分组分布对比   |

## 环节 3: 样式优化 (Style Optimization)

| 项目       | 内容                            |
| -------- | ----------------------------- |
| **核心逻辑** | 基于图表类型和数据特征自动优化视觉样式，提升可读性和美观度 |
| **输入**   | 图表规格 JSON + 原始数据              |
| **输出**   | 优化后的 Plotly Figure 对象         |
| **实现思路** | 从四个维度进行自动优化                   |

### 优化策略

**1. 配色方案**

- 分类数 ≤ 10：使用 Plotly 离散色板（plotly, D3, Set2）
- 分类数 > 10：自动切换为连续色板
- 热力图：使用 RdBu_r 双极色板展示相关系数正负

**2. 布局优化**

- 自动调整图例位置（分类多时放在右侧，少时放在底部）
- X 轴标签过长时自动旋转 45°
- 饼图自动标注百分比和标签
- 统一使用 plotly_white 模板去除多余装饰

**3. 交互增强**

- Hover 信息格式化：数值保留 1-2 位小数，显示完整维度名称
- 支持缩放、平移、框选等交互操作
- 支持图表导出为 PNG

**4. 中文适配**

- 全局设置 `font-family: Microsoft YaHei, Arial`
- 标题字号 18px，轴标签 14px
- 确保中文标签不被截断

## 环节 4: 可解释输出 (Interpretable Output)

| 项目       | 内容                          |
| -------- | --------------------------- |
| **核心逻辑** | 对生成的图表进行自然语言解读，帮助用户理解数据洞察   |
| **输入**   | 图表规格 JSON + 聚合后的数据          |
| **输出**   | 结构化文字说明（包含关键发现、趋势描述、异常提示）   |
| **实现思路** | 采用规则计算 + 模板生成的方式（可选 LLM 增强） |

### 解读维度

**1. 极值定位**

- 自动找出 Y 轴最大/最小值及其对应的 X 轴维度
- 例如："Cardio 类型卡路里消耗最高（均值 420.5），Yoga 最低（均值 185.2）"

**2. 趋势判断**

- 对折线图/散点图计算线性回归斜率
- 判断趋势方向（上升/下降/平稳）
- 例如："随着年龄增长，BMI 呈微弱上升趋势"

**3. 分布描述**

- 报告集中趋势（均值、中位数）和离散程度（标准差、极差）
- 识别明显的偏态分布
- 例如："卡路里消耗分布右偏，大部分集中在 200-400 范围"

**4. 异常提示**

- 标记离群值（超出 3σ 或 IQR 范围）
- 提示数据质量问题（如某列缺失率过高）

## 环节 5: 多轮迭代 (Multi-turn Iteration)

| 项目       | 内容                         |
| -------- | -------------------------- |
| **核心逻辑** | 保留历史交互上下文，支持用户对图表进行增量修改和对比 |
| **输入**   | 用户追加指令 + 上一轮图表规格 + 对话历史    |
| **输出**   | 更新后的图表规格 + 新图表             |
| **实现思路** | 通过 session 状态管理实现多轮对话      |

### 实现机制

**1. 上下文维护**

- 在 session 中维护 `conversation_history` 列表
- 每轮将上一次的 spec + 用户新指令一起传入 LLM
- LLM 只需修改 spec 中变化的字段（如改 chart_type 或加 color 分组）

**2. 增量指令支持**

- 支持修改图表类型："换用折线图"
- 支持添加分组："按性别分组"
- 支持筛选："只看前 10 个"
- 支持修改样式："换个配色方案"

**3. 图表历史**

- 保留每轮生成的图表，用户可滚动回溯对比
- 标注每轮使用的指令和规格

**4. 重置机制**

- 支持"重置"指令清空对话上下文
- 支持"回到上一张"撤销最近一次修改

## 整体流程图

```
用户输入自然语言指令
         │
         ▼
┌──────────────────┐     ┌──────────────────┐
│ 环节1: 数据理解   │────▶│ 环节2: 图表选择    │
│  DataFrame→画像   │     │  NL指令+画像→Spec  │
└──────────────────┘     └────────┬─────────┘
                                  │
                                  ▼
                          ┌──────────────────┐
                          │ 环节3: 样式优化    │
                          │  Spec+数据→Figure  │
                          └────────┬─────────┘
                                  │
                                  ▼
                          ┌──────────────────┐
                          │ 环节4: 可解释输出  │
                          │  Figure→文字解读   │
                          └────────┬─────────┘
                                  │
                           ┌──────▼───────┐
                           │ 用户反馈评估   │──→ 追加指令 ──→ 回到环节2（多轮迭代）
                           └──────┬───────┘
                                  │ 满意
                                  ▼
                           ┌──────────────┐
                           │ 输出最终图表   │
                           └──────────────┘
```

## 创新点

1. **双层 NL 解析**：API 智能解析 + 本地规则回退，确保任何环境可用
2. **自动数据解读**：不仅生成图表，还提供极值、趋势、异常的文字分析
3. **多轮迭代**：支持增量修改指令，无需每次重新描述完整需求
4. **零配置启动**：内置示例数据集，无需手动下载即可体验完整功能
